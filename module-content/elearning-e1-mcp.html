Module 1 van 4

Wat is MCP?

MCP staat voor Model Context Protocol. Het is een standaard waarmee Claude verbinding kan maken met externe tools en systemen. Zonder MCP werkt Claude alleen met de informatie die jij hem geeft. Met MCP kan Claude informatie ophalen uit en schrijven naar systemen die je dagelijks gebruikt.

Simpel gezegd: MCP is de brug tussen Claude en de rest van je digitale werkomgeving.

Risico's en lees- vs schrijftoegang

MCP geeft Claude toegang tot echte systemen. Dat betekent dat fouten ook echte gevolgen hebben. Als Claude een e-mail verstuurt die je niet wilde versturen, of een agenda-item verwijdert, is dat niet zomaar ongedaan te maken.

Lees- vs schrijftoegang

1

Begin met leestoegang

Claude kan informatie ophalen maar niet aanpassen. Veilig om mee te starten.

2

Test uitgebreid

Controleer of Claude de juiste informatie ophaalt en correct interpreteert.

3

Eventueel schrijftoegang

Pas als je vertrouwen hebt in de resultaten. Geef alleen schrijftoegang voor specifieke acties.

Let op

Begin altijd met leestoegang. Geef schrijftoegang alleen als je begrijpt wat Claude ermee kan doen en als je hem getest hebt. Fouten in echte systemen zijn niet zomaar ongedaan te maken.
